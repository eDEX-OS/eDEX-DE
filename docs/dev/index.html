Setup

Development environment setup

Install Rust stable, Node.js 20+, and the distro-specific system libraries required by Tauri's Linux WebKitGTK backend.

Debian / Ubuntu
sudo apt install libwebkit2gtk-4.1-dev libgtk-3-dev libssl-dev \
                 libasound2-dev libayatana-appindicator3-dev pkg-config
Fedora
sudo dnf install webkit2gtk4.1-devel gtk3-devel openssl-devel \
                 alsa-lib-devel libayatana-appindicator-gtk3-devel
Bootstrap
git clone https://github.com/eDEX-OS/eDEX-DE.git
cd eDEX-DE
npm install

Architecture

Rust commands, IPC flow, and frontend layout

Rust exposes Tauri commands for system integration, the frontend wraps them in typed IPC helpers, and Preact components consume those helpers through hooks and context providers.

High-level flow
User action → Preact component → typed IPC wrapper → Tauri invoke()
→ Rust command module → CLI / socket / PTY integration → response back to UI

Contributing

Branch naming, commit style, and PR checklist

Use conventional branches and commits to keep release automation readable:

  • Branch prefixes: feature/, fix/, docs/, chore/, refactor/
  • Commit style: Conventional Commits like feat:, fix:, and docs:
  • PR checks: run npm run typecheck and cargo check -p edex-de before you open a pull request
When adding new Tauri commands, remember to export them from src-tauri/src/commands/mod.rs, register them in src-tauri/src/lib.rs, and mirror them in ui/src/ipc/index.ts.

FAQ

Developer FAQ

Create them under ui/src/components/<Name>/, add an index file if needed, and re-export from ui/src/components/index.ts when the component is shared.
Add a Rust command module under src-tauri/src/commands/, register it in the Tauri handler list, and expose a typed wrapper in ui/src/ipc/index.ts.
For the application UI, append styles to ui/src/styles/main.css instead of replacing the existing theme pipeline. The docs site has its own isolated stylesheet under docs/assets/css/style.css.
