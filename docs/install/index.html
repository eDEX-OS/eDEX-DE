Before you install

Prerequisites

eDEX-DE is built for Linux desktops and works best on Wayland. Hyprland unlocks the full workspace and event integration, but the app will still run elsewhere with graceful fallbacks.

  • Wayland session recommended; Hyprland for full IPC features
  • WebKitGTK + GTK3 runtime libraries installed by your package manager
  • Optional integrations: wpctl/pactl, nmcli, fprintd, systemctl
If you just want the quickest path, use the packaged build for your distro. Source builds are best for contributors and testers.

Packages

Quick install tabs

Debian / Ubuntu
wget https://github.com/eDEX-OS/eDEX-DE/releases/latest/download/edex-de_amd64.deb
sudo dpkg -i edex-de_amd64.deb
sudo apt -f install
Fedora
sudo rpm -i https://github.com/eDEX-OS/eDEX-DE/releases/latest/download/edex-de.x86_64.rpm
Arch Linux
yay -S edex-de
# or: paru -S edex-de

The repository also ships packaging/install.sh for a curl-pipe bootstrap flow when you do not want to pick a package manually.

Universal installer
curl -fsSL https://raw.githubusercontent.com/eDEX-OS/eDEX-DE/master/packaging/install.sh | bash

Fedora

Install on Fedora

Install the latest RPM release asset directly, or rebuild from packaging/edex-de.spec if you prefer a local RPM workflow.

Fedora RPM
sudo rpm -i https://github.com/eDEX-OS/eDEX-DE/releases/latest/download/edex-de.x86_64.rpm

Arch Linux

Install on Arch Linux

Use the AUR helper of your choice, or build from the bundled packaging/PKGBUILD if you want to package it yourself.

Arch AUR
yay -S edex-de
# or: paru -S edex-de

Source builds

Build from source

For development builds, install Rust stable, Node.js 20+, and the distro-specific WebKit/GTK packages below.

Debian / Ubuntu dependencies
sudo apt install libwebkit2gtk-4.1-dev libgtk-3-dev libssl-dev \
                 libasound2-dev libayatana-appindicator3-dev pkg-config
Fedora dependencies
sudo dnf install webkit2gtk4.1-devel gtk3-devel openssl-devel \
                 alsa-lib-devel libayatana-appindicator-gtk3-devel
Build commands
git clone https://github.com/eDEX-OS/eDEX-DE.git
cd eDEX-DE
npm install
npm run tauri -- build
# dev mode
npm run tauri -- dev

Window manager integration

Hyprland setup

Add a startup rule and optional Meta+Space launcher bind to your Hyprland config. You can also generate this from inside eDEX-DE.

~/.config/hypr/hyprland.conf
exec-once = edex-de
bind = SUPER, Space, exec, edex-de --launcher
windowrule = pin, edex-de
windowrule = fullscreen, edex-de
windowrule = nodecor, edex-de
windowrule = noborder, edex-de
windowrule = noshadow, edex-de
windowrule = noanim, edex-de

The built-in Hyprland Config Generator saves a ready-to-source file at ~/.config/edex-de/hyprland-integration.conf.

Troubleshooting

Common install issues

Make sure you launched eDEX-DE from the same Hyprland session so HYPRLAND_INSTANCE_SIGNATURE is available. Outside Hyprland, the UI will stay up but workspace widgets are intentionally disabled.
Audio controls depend on wpctl or pactl. Install PipeWire or PulseAudio utilities and restart eDEX-DE.
Double-check the distro-specific system packages above. Tauri needs WebKitGTK, GTK3, OpenSSL, and appindicator headers to compile native bundles.
Download the newest package from GitHub Releases, or rebuild from the latest git checkout. Native package managers will replace the previous version cleanly.
